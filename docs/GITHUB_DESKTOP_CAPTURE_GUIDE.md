# GitHub Desktop 캡처 가이드

아래 세 장면은 실제 GitHub Desktop 화면이 필요한 구간입니다. 개인 계정으로 촬영할 때는 사용자 이름, 이메일, 최근 저장소 이름, 로컬 경로를 가립니다.

## 팀원 2 — 로그인 완료

- 화면: 로그인 후 GitHub Desktop 기본 창
- 반드시 보일 요소: `Current repository` 메뉴
- 강조: 계정/저장소 선택 영역
- 저장 파일: `desktop-01-signed-in.png`

## 팀원 7 — Changes 탭

- 화면: 파일 하나를 수정한 직후 Changes 탭
- 반드시 보일 요소: 파일 체크박스와 변경 줄
- 강조: Changes 목록, 수정 파일명, 초록/빨강 줄
- 저장 파일: `desktop-02-changes-tab.png`

## 공통 처리

1. 전체 화면 한 장과 강조 영역 확대 한 장을 촬영합니다.
2. 번호·박스는 누르는 요소만 감싸고 설명 텍스트는 가리지 않습니다.
3. 공개본은 `public/annotated/desktop/`에 넣기 전 개인정보를 가립니다.
4. 가이드의 `CaptureRequest` 파일명과 일치하게 저장합니다.
